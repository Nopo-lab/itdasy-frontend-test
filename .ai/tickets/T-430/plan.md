# T-430 · Sprint E 리터치 CRM + 소수점 재고

**상태:** coding  
**위험도:** 🟡 스테이징 기능 추가  
**범위:** `itdasy_backend` + `itdasy-frontend-test-yeunjun`

## 목표

forcodex.md 의 Sprint E 를 뒤에서부터 반영한다.

- 시술별 리터치 주기 저장
- 예약 완료 시 고객 다음 리터치 날짜 계산
- 재고 수량을 소수점으로 저장
- 시술별 재료 소모량 등록
- 예약 완료 시 재료 자동 차감
- 손님이 DM에서 색상·길이 등 옵션을 바꾸면 대기 중인 예약 카드에 반영

## 적용 방식

- 백엔드에 공통 처리 파일 `services/sprint_e.py` 추가
- 기존 예약 완료 처리 흐름에서 공통 처리 호출
- 시술 관리 화면에 리터치 주기와 재료 소모 연결 UI 추가
- 재고 화면에서 소수점 수량 표시·입력 가능하게 수정
- 오늘 브리핑에서 리터치 임박 손님을 보여주고 DM 초안 대기에 넣도록 연결

## 검증

- Python 문법 확인
- 변경 프론트 파일 `node --check`
- 프론트 smoke/test
- 변경 파일 중심 자동 검사
