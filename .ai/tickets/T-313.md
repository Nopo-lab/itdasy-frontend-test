# T-313 · 로그인 화면 회원가입 버튼 + 가입 폼

**상태:** pending (연준 🟢 2026-04-22)
**담당:** 오케스트레이터
**브랜치:** `main`
**위험 라벨:** `risk:auth-flow`

## 배경

현재 lockOverlay 로그인 화면에 **회원가입 진입 버튼이 없음**. 신규 사용자는 이 앱을 통해 가입할 수 없는 상태. itdasy.com 홈페이지 CTA "🚀 무료 회원가입" 은 존재하나 앱 내부엔 미구현.

## 앱 출시 후 기기 인증 관련 설명 (연준 질문 답변)

- **기기 인증만으로 가입 불가**: Apple App Store 4.8 가이드라인·Google Play 정책 모두 "이메일 또는 소셜 로그인" 필수. Device ID 단독 가입은 반려 사유.
- **권장 경로**: (1) 이메일+비번 (2) Sign in with Apple (iOS 필수) (3) Google Sign-In (Android 권장)
- **기기 인증은 "자동 재로그인" 용도**: Face ID / Touch ID / 지문으로 토큰 잠금해제 → T-317 에서 다룰 예정.

## Acceptance Criteria

- [ ] 로그인 화면에 "회원가입" 링크 추가
- [ ] `registerOverlay` 신규 — 이메일·비번·이름 + 추천 코드(선택)
- [ ] POST `/auth/register` 호출 후 자동 로그인
- [ ] 비번 8자 이상 + 영문/숫자 포함 클라이언트 검증
- [ ] 개인정보처리방침·국외이전 동의 체크박스 (법적 필수)
- [ ] 성공 시 샘플 데이터 주입 → 대시보드로
- [ ] 실패(중복 이메일 등) 시 에러 메시지

## 영향 파일

- `index.html` (registerOverlay 추가, 스타일 재사용)
- `app-core.js` (`signup()` 함수, 전환 로직)
- 버튼·링크는 기존 `.pin-btn` 스타일 재사용

## 리스크

1. 비밀번호 클라이언트 검증 실패 → 서버가 다시 검증 (안전장치)
2. 약관 동의 체크박스 없이 가입 금지 → UX 방해지만 법적 필수
3. 소셜 로그인은 별도 티켓 (Sign in with Apple T-320, Google T-321)

## Verification

1. 로그인 화면 → "회원가입" 탭 → 폼 → 제출 → 가입+자동 로그인+샘플 주입
2. 이미 쓴 이메일 시도 → 409 에러 안내
3. 약관 동의 안 하면 제출 버튼 비활성화
