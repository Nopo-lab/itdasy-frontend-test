# T-320 · Sign in with Apple (iOS 필수)

**상태:** pending (연준 🟢 2026-04-22)
**위험 라벨:** `risk:app-review-blocker`

## 배경

Apple App Store Guideline 4.8 — 타 소셜 로그인(Google 등) 제공 시 **Sign in with Apple 도 동등 제공 필수**. 미제공 시 앱 심사 반려.

## 구현 계획

1. Apple Developer Console → App ID 에 "Sign in with Apple" capability 활성화
2. Capacitor 플러그인: `@capacitor-community/apple-sign-in`
3. BE: `/auth/apple` 엔드포인트 — Apple ID token 검증 후 user 생성/로그인
4. 서버측 JWT 검증: Apple public key 캐시 + JWT exp/iss/aud 검증
5. FE: 로그인/회원가입 화면에 "🍎 Apple로 계속하기" 버튼

## 영향 파일

- `backend/routers/auth.py` — `/auth/apple` 신규
- `backend/utils/apple_verify.py` 신규
- `app-auth-social.js` 신규
- `capacitor.config.json` 앱 도메인 설정

## 리스크

1. Apple Developer Program 연 $99 필요 — 이미 가입돼 있으면 OK
2. 서버측 토큰 검증 복잡 (ES256 + JWKS)
3. 이메일 익명화 relay (`*.privaterelay.appleid.com`) 처리

## Verification

- iOS 빌드 → Apple ID 로 가입 → DB 에 Apple `sub` 와 user 매핑
- 재로그인 시 이메일 매칭
- 이메일 익명화된 경우도 정상 동작
