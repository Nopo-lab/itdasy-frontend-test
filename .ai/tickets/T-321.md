# T-321 · Google Sign-In (Android 권장 + iOS 선택)

**상태:** pending (연준 🟢 2026-04-22)
**위험 라벨:** `risk:oauth-integration`

## 배경

Android 사용자는 기본 Google 계정이 있어 원클릭 가입 선호. 또한 T-320(Apple) 과 짝 맞춰야 UX 일관.

## 구현 계획

1. Google Cloud Console → OAuth 2.0 Client (iOS / Android / Web)
2. Capacitor 플러그인: `@codetrix-studio/capacitor-google-auth`
3. BE `/auth/google` — ID token 검증 후 user 생성/로그인
4. 서버측 검증: `google-auth-library` (Python: `google.oauth2.id_token`)
5. FE: "G Google로 계속하기" 버튼

## 영향 파일

- `backend/routers/auth.py` — `/auth/google`
- `backend/utils/google_verify.py` 신규
- `backend/requirements.txt` — `google-auth`
- `app-auth-social.js` (T-320 과 공유)

## 리스크

1. `google-services.json` (Android) · `GoogleService-Info.plist` (iOS) 배포
2. Web Client ID 별도 관리 (PWA 용)
3. 이메일 중복 처리 (일반 이메일 가입 후 같은 이메일로 Google 로그인) — 기존 계정에 링크

## Verification

- Android 빌드 → Google 계정 탭 → 가입 성공
- Web PWA 에서도 Google 로그인 동작 (선택)
