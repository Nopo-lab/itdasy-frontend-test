### (c) consent_missing 메시지 변경
- 현재: "페르소나 탭 하단 AI 처리 동의를 먼저 체크해주세요"
- 페르소나 탭 삭제 → 동의 체크박스 위치 이전 필요? 
- **확인 필요**: 동의 체크박스가 페르소나 탭 안에만 있었는지, 글쓰기 온보딩 팝업 안에도 있는지

**→ 미확정 항목**: `showOnboardingCaptionPopup` 내부에 AI 처리 동의 체크박스가 포함돼 있는지 확인 후, 없으면 **추가** 필요. 있으면 메시지만 교체. 이 확인은 승인 후 실제 작업 단계에서 `app-caption.js:206~` 읽고 판단.

### (d) index.html L184 `header-persona` 제거 영향
- 홈 헤더의 페르소나 관련 영역. 삭제 후 헤더 레이아웃 체크 필요.
- CSS (`style-home.css`) 에 `header-persona` 클래스 참조 있는지 함께 정리.

### (e) initPersonaTab 참조 제거

## 4. 미확정 항목 (승인 시 원영님 확인 요청)

1. **동의 체크박스 위치**: 현재 AI 처리 동의 체크박스가 페르소나 탭에만 있으면 신 온보딩 팝업에 추가해야 할 수 있음. → 승인 후 작업 중 확인 → 필요 시 추가 코드 발생 (plan 예상 diff +10줄 증가 가능)
2. **header-persona 영역**: 홈 상단에 노출되는 요소인지 확인 필요. 노출 중이면 UX 변화 있음.
3. **개별 커밋 분리 vs 단일 커밋**: plan §8 에 5~6 커밋 제안. 원영님이 "단일 커밋" 선호하시면 조정.
4. **노쇼 서버 응답**: 백엔드가 여전히 `status: 'no_show'` 를 보내면 클라이언트에서 "알 수 없음" 으로 표시됨. 이대로 두어도 괜찮은지.
