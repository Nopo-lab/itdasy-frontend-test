# T-317 · 생체 인증 자동 재로그인 (Face ID / 지문)

**상태:** pending (연준 🟢 2026-04-22)
**담당:** 오케스트레이터
**위험 라벨:** `risk:native-integration`

## 목표

첫 로그인 후, 앱 재실행 시 **Face ID / 지문** 인증으로 즉시 진입. 비번 재입력 없이.

## 기술 스택

- **Capacitor 플러그인**: `@aparajita/capacitor-biometric-auth` (무료, 활발 유지)
  - iOS: LocalAuthentication (Face ID / Touch ID)
  - Android: BiometricPrompt (지문 / 얼굴)
- **PWA 폴백**: WebAuthn Platform Authenticator (Chrome/Safari 최신)
- **토큰 저장**: iOS Keychain + Android Keystore (플러그인 제공)

## 구현 계획

1. `npm install @aparajita/capacitor-biometric-auth` (Capacitor 프로젝트 환경)
2. `app-biometric.js` 신규 — 로그인 직후 "지문·Face ID 로 다음부터 빠르게 열기" 제안 팝업
3. 동의 시 토큰을 플러그인 secure storage 에 저장
4. 앱 실행 시 토큰 있으면 → 생체 인증 → 복호화 → 자동 로그인
5. 인증 실패 3회 시 이메일+비번 폴백

## 영향 파일

- `app-biometric.js` (신규)
- `app-core.js` — 자동 로그인 플로우 분기
- `package.json` (Capacitor env)
- `capacitor.config.json`
- `index.html` 스크립트 등록

## 리스크

1. Capacitor iOS/Android 네이티브 빌드 필요 — 웹 PWA 에선 미동작 (WebAuthn 폴백 제공)
2. 플러그인 설치 시 iOS 빌드 설정 (Info.plist `NSFaceIDUsageDescription` 추가)
3. 인증 실패 시 사용자 혼란 → 명확한 에러 + 이메일 폴백

## Verification

- 로그인 직후 "지문 등록하시겠어요?" 팝업 → 예 → 다음 앱 실행 시 지문 진입
- 지문 실패 3회 → 이메일 로그인 폼
- 기기 설정에서 생체 미등록 상태 → 팝업 안 뜸
